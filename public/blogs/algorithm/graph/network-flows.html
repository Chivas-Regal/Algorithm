    将图分为两个点集，源点 sss 位于点集 SSS 中，汇点 ttt 位于点集 TTT 中，割的大小为 SSS 到 TTT 的可行边数量（对于一个点集内的点可以不连通）

    割的大小为 222

    最多有多少条不相交路径

    转移成判定问题：判定是否存在 kkk 条不相交路径
    k=1k=1k=1 就是连通性证明
    k=2k=2k=2 任意一对 STSTST 割大小都不小于 222，因为 s→ts\to ts→t 的路径必有至少一条边会从 SSS 到 TTT ，若存在一对 STSTST 割大小为 111 ，那么这条路必经，就不存在两条不想交路径
    k=...k=...k=...
    这就是 最小割问题（割掉最少的边，使图不连通）

    # dicnic算法解决

    对于带权图，我们跑完一遍增广路之后得到的残留网络中跑的边要反向，那么我们可以直接让这条路上每一条的反边加上我们跑出来的流量，正边减去流量。
    而增广路不能绕环，所以 dicnicdicnicdicnic 算法先对残留网络进行分层，得到每一个点到 sss 的最短路径 deepdeepdeep ，增广路的要求是这条路径上的 deepdeepdeep 必须递增且连续的。

    在分层图上找增广路的流程就是：从 uuu 出发，若边有流量且 vvv 比 uuu 高一层，那么继续向 vvv 找。
    这样的复杂度有点说不过去了，毕竟这算是 dfsdfsdfs 暴力跑图了，这里有三个优化。

    • 多路增广
    • 当前弧优化
    • 炸点

    多路增广是说如果我们在跑完 uuu 的第一条边之后前面的流量还没有流满，那么就用剩下的流量空间去跑第二条边、第三条边...
    当前弧优化是说如果我们现在跑的不是第一条边，说明第一条边已经尽自己所能去流了，在这次分层图上它之后就没什么用了，就直接让下一条边作为 uuu 的第一条边
    炸点是说若当前点往下流的全都是 000 ，那么这个点在这次分层图上没有往后面跑的意义，将其 deepdeepdeep 设为一个走不到的数就可以保证之后永远不走它了

    这里是代码演示：

    inline void bfs (int S, int T) {...} // 就是求一个边权为 1 的最短路，没有必要演示了

// 参数：
// 当前点为 u，到 u 之前剩余流量为 fl

// 变量：
// 从 u 出发的总流量为 f
// 目标节点为 v，<u,v> 反边为 edge[i ^ 1]
inline int dfs (int u, int fl) { 
        if (u == T) return fl;
        int f = 0; 
        for (int i = head[u]; i && fl; i = edge[i].nxt) { // 多路增广
                head[u] = i; // 当前弧优化
                int v = edge[i].to;
                if (edge[i].flow && deep[v] == deep[u] + 1) {
                        int x = dfs(v, min(fl, edge[i].flow));
                        edge[i].flow -= x;
                        edge[i ^ 1].flow += x; 
                        fl -= x;
                        f += x;
                }
        }
        if (!f) deep[u] = -2; // 炸点
        return f;
}
inline int dicnic (int S, int T) {
        int ret = 0;
        while (bfs(S, T)) ret += dfs(S, 0x3f3f3f3f);
        return ret;
}

    # 费用流

    费用流说的都是 “最..费用最..流”
    这里我们建边时要加一个费用信息，同时反边这个纠正边意味着我们不选择这些费用，就将反边的费用设为正边费用的相反数即可。
    而这样最短路就不能使用普通的 bfsbfsbfs 来求了，而且存在负数的 costcostcost 费用，所以要跑一个 spfaspfaspfa 代替 bfsbfsbfs
    然后在这样的最短路图上继续跑 dfsdfsdfs 即可

    # 无源汇可行流

    ClC_lCl​ 为容量下界， CuC_uCu​ 为容量上界，fff 为流量
    要求 Cl(u,v)≤f(u,v)≤Cu(u,v)C_{l(u,v)}\le f_{(u,v)}\le C_{u(u,v)}Cl(u,v)​≤f(u,v)​≤Cu(u,v)​
    转化为 0≤f(u,v)−Cl(u,v)≤Cu(u,v)−Cl(u,v)0\le f_{(u,v)}-C_{l(u,v)}\le C_{u(u,v)}-C_{l(u,v)}0≤f(u,v)​−Cl(u,v)​≤Cu(u,v)​−Cl(u,v)​
    每条边的可行流就是在这个图上的边的流量加上其下界

    但是碍于流量守恒，可能一个点的入边下界和小于出边下界和，减去的不一样多，在减过的图守恒但是补回来就不守恒了。
    这里构建超级源点 SSS ，超级汇点 TTT ，如果 uuu 的入边下界和过大，补充一条 <S,u><S,u><S,u> 流量为入边下界和减去出边下界和，如果出边下界和过大，补充一条 <u,T><u,T><u,T> 流量为出边下界和减去入边下界和即可。
    当 SSS 出发的边全部流满，说明满足了流量守恒，才有可行流。

    # 有源汇的最大可行流

    原源汇为 s,ts,ts,t ，新源汇为 S,TS,TS,T ，建图和上面一样，但要保证流量守恒的话，还要加上 <t,s><t,s><t,s> 容量为 ∞\infty∞
    令新图 s→ts\to ts→t 的最大可行流为 fff ，S→TS\to TS→T 的一个满流为 f′f'f′ ，在找完 S→TS\to TS→T 满流的残留网络上再找 s→ts\to ts→t 的可行流为 fs→t′f'_{s\to t}fs→t′​ （删掉 t→st\to st→s 的边）
    由 fff 中 SSS 出边与 TTT 入边均已流满， f′f'f′ 同样，故 f−f′f-f'f−f′ 一定不包含 SSS 出边与 TTT 入边，那么让 fs→t′f'_{s\to t}fs→t′​ 不守恒的因素 S,TS,TS,T 已经消失，则 f′f'f′ 与 fs→tf_{s\to t}fs→t​ 一一对应，所以 fs→t′+f′=ff'_{s\to t}+f'=ffs→t′​+f′=f

    故流程就是，按无源汇建图后，添加一条 t→st\to st→s 的边，跑完 dicnicdicnicdicnic 得到 S→TS\to TS→T 的可行流下 s→ts\to ts→t 的最大流，然后在残余网络中删去 <t,s><t,s><t,s> 的边（正反），再跑一遍 s→ts\to ts→t 的最大流。
    两次最大流相加就是答案。
