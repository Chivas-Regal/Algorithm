    Chivas-Regal 数学

    # 博弈论

    # 特点

    • 两个以上玩家
    • 状态为有限的集合
    • 双方轮流操作
    • 具有规则
    • 有限次操作结束
    • 有胜负
    • 两人足够聪明

    # 必定点

    必胜（败）点：在该点进行操作的人必胜（败）

    # 属性

    1.1.1. 所有终结点为必败点
    2.2.2. 每个必胜点至少有一个操作进入必败点
    3.3.3. 必败点只能走入必胜点

    # 图示

    # 巴什博弈

    222 人， nnn 枚石子，双方轮流选取，不能拿或且不可拿超过 xxx 枚，没石子可拿的人就输了
    这个问题由于两个人一定足够聪明，他们会有一种“跟着选”的策略
    即：如果当前石子数为 x+1x+1x+1 的倍数，那么如果一个人拿了 aaa 个，那么第二个人只要跟着拿 (x+1)−a(x+1)-a(x+1)−a 个就一定可以赢

    那么如果 nnn 是 x+1x+1x+1 的倍数，那么第一个人就一定会输
    同时如果不是的话，第一个人可以把这 nnn 个石子变成 x+1x+1x+1 的倍数，他就一定可以获胜了

    # 尼姆博弈

    222 人，多堆石子，每个人选取其中一堆石子取走任意个石子

    三堆石子分析
    (0,0,0)(0,0,0)(0,0,0) 必败
    (0,0,x)(0,0,x)(0,0,x) 必胜
    (0,1,1)(0,1,1)(0,1,1) 必败
    (0,x,x)(0,x,x)(0,x,x) 必败
    (x,y,z)(x,y,z)(x,y,z) ???

    # 尼姆和

    按位的异或（上面是 x⊕y⊕zx\oplus y\oplus zx⊕y⊕z)
    其中 000 必败，否则必胜

    证明: 例如三堆石子为 13,12,813,12,813,12,8
    13=1101212=11002⊕8=10002nim_sum=10012=9\frac{\begin{aligned}13&=1101_2\\12&=1100_2\\\oplus\;\;\;\;8&=1000_2\end{aligned}}{nim\_sum=1001_2=9}nim_sum=10012​=91312⊕8​=11012​=11002​=10002​​​


    如果有方法令 999 变成 000 则必胜（属性 ②②② ）
    可以令第一个数变成 010001000100 ，则
    13=0100212=11002⊕8=10002nim_sum=00002=0\frac{\begin{aligned}13&=0100_2\\12&=1100_2\\\oplus\;\;\;\;8&=1000_2\end{aligned}}{nim\_sum=0000_2=0}nim_sum=00002​=01312⊕8​=01002​=11002​=10002​​​
    朴素做法：结果从前往后，第一个 111 上面一定是有奇数个 111 ，选择一行该位置为 111 的，这个位置变成 000 ，后面再遇到选取同行并取反
    ∴\therefore∴ 满足


    如果任意方法都会让结果非 000 则必败（属性 ③③③ ）
    本身已经是 000000000000 平衡了，即任意位置都有偶数个 111
    操作后必定会改变其中一列的 111 的个数成为奇数个，打破平衡使结果非 000 ，必败
    ∴\therefore∴ 满足

    # SGSGSG 函数

    xxx 节点的 sgsgsg 值是不等于 xxx 后继节点的 sgsgsg 值的最小非负整数
    即： sg(x)=mex({sg[sonx]})sg(x)=mex(\{sg[son_x]\})sg(x)=mex({sg[sonx​]})

    其中 000 必败，否则必胜

    证明： 看巴什博弈(最多可以拿三个)

    终结状态 sgsgsg 必 000 ，非零必胜 （属性 ①①① ）
    能转移到 000 的均必胜（属性 ②②② ），在 sgsgsg 内必定不为 000
    不能转移到 000 ，即必须转移到正整数，本身一定是 000 ，必败（属性 ③③③ ）

    # 222

    给定规则数 nnn ，即可以拿走 a[1～n]a[1～n]a[1～n] 个石子
    给定情况数 mmm ，每种情况有 ooo 堆牌，每堆 x[1～o]x[1～o]x[1～o] 张，讨论每种情况的先手输赢

    程序采用记忆化搜索

    int n, m;   // 规则数，情况数
int a[100]; // 规则

int sg[10001];
inline int SG ( int all ) {
        bool vis[101] = {0};
        for ( int i = 0; i < n; i ++ ) {
                int remain = all - a[i]; // 剩余的石子数
                if (remain < 0) break;   // 少于0个不存在
                if ( sg[remain] == -1 ) sg[remain] = SG(remain); // 继续递归子状态的sg值
                vis[sg[remain]] = 1;     // 子状态sg值标记一下
        }
        for ( int i = 0; ; i ++ ) if ( !vis[i] ) return i; // sg定义
}

int main () {
        cin >> n; 
        for ( int i = 0; i < n; i ++ ) cin >> a[i];
        
        sort(a, a + n); // 可以在sg内部选取拿走的石子数量上进行 break 优化
        memset(sg, -1, sizeof sg);
        sg[0] = 0;

        cin >> m; while ( m -- ) {
                int res_sg = 0;

                int o; cin >> o; while ( o -- ) {
                        int x; cin >> x;
                        if ( x == -1 ) sg[x] = SG(x);
                        res_sg ^= sg[x];
                }

                if ( res_sg == 0 ) cout << "先手必败" << endl;
                else cout << "先手必胜" << endl;
        }
}
