        priority_queue<node> pque;
        pque.push({1, 0});

    # 🔗

    # 💡

    两个限制，即模块111和模块222的完成数要达到 mmm
    但是没有对应的价值，那么我们可以令一个模块的完成数作为价值，一个作为限制
    那么开 dpdpdp
    dp[i][j]dp[i][j]dp[i][j] 表示前 iii 个人完成 jjj 个模块111下最多能完成多少个模块222
    由于时间固定出每个物品的价值（完成物品体积个模块111后又能完成多少个模块222），那么就是枚举时间，枚举物品，枚举背包容积，再枚举物品容积
    这是一个 O(n4)O(n^4)O(n4) 的复杂度
    但是考虑一下随着时间增大， dp[n][m]dp[n][m]dp[n][m] 肯定是单调递增的，那么时间使用二分答案判断 dp[n][m]dp[n][m]dp[n][m] 是否 ≥m\ge m≥m

    const int N = 110;
const int M = 110;

int n, m;
int a[N], b[N];

int dp[N][M];

inline bool Check ( int tim ) {
        memset(dp, -0x3f3f3f3f, sizeof dp);
        dp[0][0] = 0;
        for ( int i = 1; i <= n; i ++ ) {
                for ( int j = 0; j <= m; j ++ ) {         // 一共做了 j 个模块1
                        for ( int k = 0; k <= j; k ++ ) { // 第 i 个人做 k 个模块1

                        }
                }
        }
        return dp[n][m] >= m;
}

                int mid = (l + r) >> 1;
                if ( Check(mid) ) r = mid - 1;
                else l = mid + 1;
        }

    平均数
    问题在于： 在不同选中数量下，一个数对平均数的贡献是不同的
    但是我们求总值是很好求的
    这个就奠定了我们可以给出一个平均数 ave 来判断能否获得比这个平均数更大的值
    每一个数的贡献就变成了 x-ave
    只要最终贡献大于 0 ，我们就可以说能造出来
    通过这个编写一个 check ，过程中采用 dpdpdp 的解法去递推，判断最终结果即可

    如果这个差大于 000 ，那么就说明我们可以造出比它更高的中位数

                if ( i.b == 1 ) mst.insert(i.pos);

                        if ( *id <= q[i.pos].b - len ) return true;  // 这个也能满足子集合的区间右端点
                }
        }
        return false;
}

        }

}
